# MASTER INDEX DE VOICESHOP

## 1. Objetivo

Define el punto de entrada, orden de lectura, módulos, navegación por roles y reglas de cierre de la especificación completa.

## 2. Propósito

El Master Index es el punto de entrada oficial a toda la especificación.

Debe permitir localizar módulos, documentos, dependencias, ADR, glosario, trazabilidad y guías operativas.

## 3. Orden de lectura

Primero: visión general, alcance y principios.

Segundo: arquitectura, dominio y contratos.

Tercero: voz, LLM, canales e integraciones.

Cuarto: administración, operación y despliegue.

Quinto: ADR, glosario, trazabilidad y catálogos maestros.

## 4. Módulos

Core y arquitectura.

Dominio comercial.

Conversación, voz y Realtime.

LLM y Tool Proposals.

Canales e integraciones.

Administración y BackOffice.

Operación y despliegue.

ADR, glosario y consolidación.

## 5. Navegación por rol

Arquitecto: ADR, arquitectura, dominio, integraciones y operación.

Desarrollador: contratos, Commands, eventos, errores y pseudocódigo.

QA: criterios, matriz, testing, casos adversariales y evidencia.

DevOps/SRE: despliegue, observabilidad, capacity, backup, DR y runbooks.

LLM implementadora: Master Index, glosario, ADR, reconstruction guide y módulo específico.

## 6. Convenciones

Los documentos se ordenan por prefijo numérico.

Los nombres de archivo son normativos.

Las regeneraciones sustituyen por completo a la parte anterior.

## 7. Cierre

La versión final debe incluir manifest, checksums, reconstruction guide y findings.

El historial del chat no es parte del producto final.

## 11. Pseudocódigo

```text
function finalize_documentation(packages):

    resolved_packages = resolve_regenerations(packages)
    extracted_files = []

    for package in resolved_packages:
        validate_zip_integrity(package)
        validate_package_file_count(package, maximum=3)

        files = extract_to_isolated_directory(package)

        for file in files:
            validate_safe_filename(file)
            validate_utf8(file)
            validate_header(file)
            validate_document_id(file)
            validate_end_marker(file)

            metadata = calculate_file_metadata(
                file,
                size_bytes=true,
                line_count=true,
                sha256=true
            )

            extracted_files.append(metadata)

    detect_duplicate_filenames(extracted_files)
    detect_duplicate_document_ids(extracted_files)
    detect_missing_document_ranges(extracted_files)

    ordered = sort_by_document_prefix(extracted_files)

    references = extract_cross_references(ordered)
    validate_cross_references(references, ordered)

    findings = run_consistency_review(
        files=ordered,
        references=references
    )

    if findings.has_critical:
        reject(FINALIZATION_BLOCKED)

    manifest = build_versioned_manifest(ordered)
    master_index = build_master_index(ordered, manifest)
    checksums = build_checksum_file(ordered)

    freeze_final_version(
        master_index,
        manifest,
        checksums,
        findings
    )

    return build_finalization_result(
        master_index,
        manifest,
        checksums,
        findings
    )
```
