# ÍNDICE DE ESPECIFICACIÓN FUNCIONAL DE VOZ

## 1. Objetivo

Esta carpeta define el comportamiento funcional completo del subsistema
de voz en tiempo real de VoiceShop.

El objetivo es permitir que un Cliente:

- inicie una sesión de voz;
- hable de forma natural;
- interrumpa una respuesta;
- reciba audio;
- use las mismas reglas comerciales que en texto;
- cambie entre voz y texto;
- se recupere de fallos;
- mantenga contexto;
- ejecute herramientas únicamente mediante el backend autorizado.

La voz es un canal de interacción.

La voz no es el dominio.

La voz no puede modificar precios, stock, pedidos, pagos o permisos por
sí misma.

## 4. Flujo funcional general

1. Cliente activa voz.
2. Canal solicita permiso de micrófono.
3. Backend valida Session ID.
4. Backend valida políticas y límites.
5. Backend obtiene autorización efímera.
6. Cliente establece conexión Realtime.
7. VoiceSession entra en READY.
8. Cliente habla.
9. El audio se transmite en chunks.
10. Se detecta inicio y fin del turno.
11. Se genera transcripción.
12. El backend procesa intención y entidades.
13. El dominio ejecuta el caso de uso.
14. Se construye una respuesta textual oficial.
15. Se genera audio.
16. El Cliente escucha.
17. Si el Cliente interrumpe, se cancela la reproducción.
18. Se inicia un nuevo turno.
19. La sesión puede cerrarse o degradarse a texto.

## 7. Reglas transversales

RULE-VOICE-INDEX-001

La API Key permanente nunca se entrega al navegador.

RULE-VOICE-INDEX-002

Toda sesión de voz está vinculada a una Session válida.

RULE-VOICE-INDEX-003

El audio no entra al dominio comercial.

RULE-VOICE-INDEX-004

Toda Tool Proposal de voz se valida igual que en texto.

RULE-VOICE-INDEX-005

Una transcripción de baja confianza no ejecuta acciones irreversibles.

RULE-VOICE-INDEX-006

Toda confirmación sensible debe ser explícita.

RULE-VOICE-INDEX-007

Una respuesta interrumpida no se considera escuchada completamente.

RULE-VOICE-INDEX-008

El cambio de voz a texto conserva contexto.

RULE-VOICE-INDEX-009

El fallo de voz no debe corromper la sesión conversacional.

RULE-VOICE-INDEX-010

La sesión de voz debe poder cerrarse limpiamente.

RULE-VOICE-INDEX-011

Los recursos deben liberarse tras expiración o desconexión.

RULE-VOICE-INDEX-012

Toda conexión debe poseer identidad, Session ID y VoiceSession ID.

RULE-VOICE-INDEX-013

Todo turno debe poseer Turn ID.

RULE-VOICE-INDEX-014

Toda interacción debe ser trazable.

RULE-VOICE-INDEX-015

La privacidad del audio debe ser explícita.

## 11. Objetivos de calidad

El subsistema debe ser:

- de baja latencia;
- interrumpible;
- recuperable;
- seguro;
- auditable;
- independiente del proveedor;
- compatible con texto;
- tolerante a ruido;
- resistente a duplicados;
- respetuoso de privacidad.

## 12. Casos prohibidos

PROHIBITED-VOICE-INDEX-001

Entregar una API Key permanente al Cliente.

PROHIBITED-VOICE-INDEX-002

Permitir que el proveedor de voz modifique el dominio.

PROHIBITED-VOICE-INDEX-003

Ejecutar una Tool Proposal parcial.

PROHIBITED-VOICE-INDEX-004

Confirmar pedido con transcripción ambigua.

PROHIBITED-VOICE-INDEX-005

Cobrar sin confirmación oficial.

PROHIBITED-VOICE-INDEX-006

Considerar escuchado audio no reproducido.

PROHIBITED-VOICE-INDEX-007

Perder contexto al cambiar a texto.

PROHIBITED-VOICE-INDEX-008

Mantener sesiones huérfanas.

PROHIBITED-VOICE-INDEX-009

Guardar audio indefinidamente sin política.

PROHIBITED-VOICE-INDEX-010

Ignorar la expiración de autorización.
