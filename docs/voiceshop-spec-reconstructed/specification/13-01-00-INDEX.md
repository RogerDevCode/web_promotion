# CONVERSATION ENGINE

----------------------------------------------------------------------
1. Objetivo
----------------------------------------------------------------------

Esta carpeta especifica completamente el motor conversacional de
VoiceShop.

El objetivo es describir el comportamiento funcional desde que un
mensaje entra al sistema hasta que una respuesta es entregada al
cliente.

No describe tecnologías específicas.

No describe librerías.

No describe modelos concretos de IA.

Describe únicamente el comportamiento esperado.

----------------------------------------------------------------------

2. Alcance
----------------------------------------------------------------------

El motor conversacional es responsable de:

• recepción de mensajes

• validación inicial

• normalización

• recuperación de contexto

• recuperación de memoria

• interpretación

• extracción de entidades

• recuperación de slots

• construcción del Prompt

• invocación de herramientas

• validación de resultados

• generación de respuesta

• aclaraciones

• manejo de errores

• auditoría

• observabilidad

----------------------------------------------------------------------

5. Objetivos de calidad
----------------------------------------------------------------------

El motor conversacional debe ser:

• determinista cuando sea posible

• explicable

• auditable

• idempotente

• escalable

• independiente del proveedor de IA

• resistente a Prompt Injection

• resistente a mensajes duplicados

• resistente a fallos parciales

• recuperable tras reinicios
